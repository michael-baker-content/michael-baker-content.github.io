## Why a Structured Path

There is a lot of noise in frontend learning resources. Having a structured curriculum removes the decision fatigue of figuring out what to learn next and lets you focus on actually learning.

## HTML and CSS Foundations

The fundamentals turned out to be deeper than I expected. CSS in particular has a lot of nuance around specificity, the box model, and layout systems like Flexbox and Grid that rewards careful study rather than just copying patterns.

## JavaScript: Where It Gets Interesting

Moving into JavaScript is where the curriculum starts to feel genuinely powerful. The jump from static pages to dynamic interfaces is significant, and understanding the event loop and asynchronous patterns is essential for anything beyond toy examples.

## React

React's component model took a little time to internalize but once it clicked it changed how I think about UI structure generally. Thinking in terms of state and derived rendering is a useful mental model even outside of React specifically.

## What's Next

The path continues into Node.js, Express, and eventually Next.js. I'm particularly curious about Next.js given how much it has become a default choice for React-based projects.
